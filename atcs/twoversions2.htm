2: Create the Topic Projects

Next you need to create an entire Help & Manual project for each version of the topic. In this tutorial the two projects are called Full_Version.hmxz and Demo_Version.hmxz (it is best to use the HMXZ single-file compressed format for these projects since they only contain a single topic).

Both child projects will contain only one topic with the topic ID ReadMe and will also have defined custom text variables. The first child only gets merged when you publish with the include option Full Build, the second only when you publish with the include option Demo Build.

The result is that we never have both help projects together (if you attempt to do so, you will get conflicting topic IDs).

Step 1: Create the topic projects

1.Create two new projects and save them in the project directory of your main project.

2.In each project create just one topic. The topic ID of this project must be identical in both projects. In this example we are using the ID ReadMe.

3.If you are using styles import the styles of your main project to the new projects. To do this select Styles > Edit Styles in the Write tab and then select Copy Styles From to copy the styles from your main project.

4.Enter the content of the topics in the two projects and save the projects.

Step 2: Add individual variables to the projects (Optional)

If you add variables to the projects you can use them in your entire project and their definitions will change automatically depending on which child project you are using. You can then also use the variables in the master project and in any other projects merged into the final help.

1.Open each project and define your variables. Only use variables that have not been defined in the master project or any other project you are going to merge into the master project at the same time!

2.When you use the variables in the master project or other projects in your help system you must type them in manually, using the syntax <%VARIABLE_NAME%>.

This method should only be used if you are going to be swapping out child projects anyway. For  more efficient methods for redefining variables see the chapters on Command Line Options and Variables in the main Help & Manual help (in the More Advanced Procedures section).

Note that you can also redefine variables for individual topics in the Topic Options tab behind the main editor window.

Step 3: Add the child projects to your main project

1.Open your main project and select the position in your TOC where you want the topic to go.

2.Select Add Topic in Project > Manage Topics and select the option Include a Help Project.

3.Select the option Merge content on publishing – runtime merging would not work because you want the topic from the external project to be part of the project.

4.Select the first project you created in Step 1 above, then repeat for the second project, so that both projects are inserted directly above one another in the TOC.

You can see what it's supposed to look like in this tutorial project. Here the two projects Full_Version.hmxz and Demo_Version.hmxz have been inserted at the end of this chapter.

Step 4: Set the build options for the child projects

In your main project, select the main project entry of the first child project in the TOC. Then select Change > Include in Builds in Project > Manage Topics and set the appropriate the build option for the project. (You can also right-click on the main project entry to access the build options.)

Repeat for the other child project (and for any others you may have created).

Important:
ONLY select the one custom build option for each topic, DON'T select any additional format-based build options. Since the build options use OR logic the projects would then always be included, defeating the object of this exercise.
