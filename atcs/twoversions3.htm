3: Link to the Topic

The next step is to create your links to the topic. Since the topic doesn't exist in the current project you are going to have to do this manually:

Procedure:

1.Make a note of the ID of the topic (it's ReadMe in this tutorial).

2.Proceed as you normally would to create a hyperlink, but enter the topic ID manually in the Topic ID: field. Help & Manual will warn you that the link doesn't exist. That's OK, just ignore the warning.

3.When you have created the hyperlink it will be highlighted in red as in the example below because Help & Manual can't find its target. That is also OK, because the target will be available once you have published your project.

This is a demo link to the Read Me topic

It doesn't matter which version you publish, this link will always point to the Read Me! topic, because both versions have the same ID.

Why not just create a normal external link?

If you're an experienced Help & Manual user you might be wondering why we don't just specify one of the embedded project files in the Insert Link dialog, then we could select the target project normally.

Unfortunately, that wouldn't work. In HTML Help the link would point to a topic in an external help file and we are merging the target file into the main project. Remember, we are using publish-time merging, that is why we need to use this trick in the first place! It also wouldn't work in Browser-based Help or other formats, because here no "external help file" is available.
