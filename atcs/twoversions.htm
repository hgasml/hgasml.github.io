Two Versions of the Same Topic

Tutorial level: Advanced

Requirements: You should be familiar with working with modular projects and text variables before trying to use the methods described in this chapter.

There are situations where you may want to have two completely different versions of the same topic – for example for two different versions of your program for different user groups. If you also have links to the topic from other topics you have a problem, because Help & Manual won't allow you to create two topics with the same topic ID.

Of course, you can create two versions of the topic text within the same topic and enclose them in conditional text tags. This is very simple and it works, but it can be a little difficult to manage.

The trick: Create two topic versions as external H&M projects

The solution is to create the two versions of the topics as separate Help & Manual projects, both with the same topic ID, and insert them in your project as modules.

Since modules can be included and excluded in the same way as topics you can then use conditional output to include only the version you want. Since both topics have the same topic ID, links to the topic from other topics will always work.

Of course, you can't include both versions! If you do you will get errors because they both have the same topic ID!

Which output formats you can use this with:

In theory, you can use this method for all output formats. However you only really need it for electronic output formats containing hyperlinks (including PDF, if you have activated hyperlinks there).
